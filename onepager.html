Comment fonctionne le chat Streamlit + WebSocket

Ce projet sépare l'interface utilisateur et le transport temps réel. Streamlit affiche l'écran de chat, mais les messages transitent en direct par un serveur WebSocket FastAPI lancé avec Uvicorn.

Interface

Streamlit rend la page, lit l'URL WebSocket depuis le secret et injecte un composant HTML + JavaScript pour ouvrir la connexion.

Transport

FastAPI maintient une liste de connexions par salon et diffuse chaque message à tous les navigateurs du salon.

Usage

Le lancement standard se fait via start.ps1, puis deux onglets avec le meme salon suffisent pour voir les messages en temps reel.

Architecture en clair

1
Le script de demarrage lance les deux services

Le fichier start.ps1 lit .streamlit/secrets.toml si present, recupere le port WebSocket puis ouvre Uvicorn et Streamlit dans deux fenetres PowerShell.

2
Le navigateur ouvre Streamlit

L'utilisateur charge l'application Streamlit puis choisit un salon et un pseudo. L'URL du serveur WebSocket vient du secret, sans saisie manuelle dans l'interface.

3
Streamlit injecte un mini client chat

Le fichier app.py génère une page HTML embarquée avec JavaScript. Ce script crée une WebSocket vers /ws/{room}.

4
Le serveur rattache la connexion à un salon

Le serveur websocket_server.py accepte la connexion, stocke le client dans une liste propre au salon, puis annonce l'arrivée.

5
Chaque message est diffusé

Quand un client envoie un JSON, le serveur le redistribue à tous les clients connectés dans le même salon.

Commande recommandee
.\start.ps1
Demarrage manuel
uvicorn websocket_server:app --host 0.0.0.0 --port 9876 streamlit run app.py --server.port 8501
Avec la configuration actuelle, .streamlit/secrets.toml pointe vers ws://localhost:9876/ws. Si ce fichier n'existe pas, l'application retombe automatiquement sur ws://localhost:8765/ws.

Cycle d'un message

Etape A

L'utilisateur tape un message dans la zone de saisie du composant HTML et clique sur Envoyer ou appuie sur Entrée.

Etape B

Le JavaScript envoie un JSON de ce type au serveur : { type: "message", username, message }.

Etape C

FastAPI lit ce JSON via la WebSocket, reconstruit un payload propre puis exécute une diffusion dans la room concernée.

Etape D

Chaque navigateur reçoit l'événement WebSocket, parse le JSON et ajoute la bulle dans la bonne couleur selon l'auteur du message.

Navegateur 1 -> WebSocket send(JSON) -> serveur FastAPI /ws/{room} -> broadcast a toutes les connexions de la room -> Navegateur 1 et Navegateur 2 recoivent le message

Role des fichiers

app.py

Configure Streamlit, lit l'URL WebSocket depuis le secret ou le fallback par defaut, affiche les champs de connexion utiles et injecte le client HTML/JavaScript de chat.

websocket_server.py

Expose le serveur FastAPI, gère les connexions par salon, diffuse les messages et annonce les entrées/sorties des navigateurs.

.streamlit/secrets.toml

Optionnel. Permet de surcharger l'URL WebSocket sans modifier le code. Ici, il configure ws://localhost:9876/ws.

start.ps1

Script de lancement Windows. Il lit le secret si disponible, deduit le port WebSocket, puis demarre Uvicorn et Streamlit ensemble.

requirements.txt

Déclare les dépendances nécessaires : Streamlit, FastAPI et Uvicorn.

Ce qu'il faut retenir

  • Streamlit ne transporte pas les messages lui-même : il sert surtout de conteneur d'interface.
  • Le temps réel est assuré par la route WebSocket /ws/{room} du serveur FastAPI.
  • Le demarrage le plus simple sous Windows est .\start.ps1, qui garde le port Uvicorn synchronise avec le secret.
  • L'utilisateur n'a pas a saisir l'adresse WebSocket dans l'interface: elle est lue depuis `.streamlit/secrets.toml`.
  • Le nom de salon sert de clé pour regrouper les navigateurs dans la même conversation.
  • Si un navigateur se déconnecte, le serveur le retire de la liste du salon et diffuse un message système.
